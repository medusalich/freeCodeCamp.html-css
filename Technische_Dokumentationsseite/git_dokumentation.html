Einleitung

Git ist ein weit verbreitetes Versionskontrollsystem, das Entwicklern hilft, Änderungen an ihrem Code zu verfolgen und zu verwalten. Es wurde ursprünglich von Linus Torvalds, dem Schöpfer des Linux-Betriebssystems, entwickelt. Mit Git können mehrere Personen gleichzeitig an einem Projekt arbeiten, ohne sich gegenseitig in die Quere zu kommen.

Das Besondere an Git ist, dass es eine lokale Kopie des gesamten Projekts auf dem Computer eines Entwicklers speichert. Das bedeutet, dass man auch offline arbeiten kann und alle Änderungen, die man vornimmt, lokal festgehalten werden. Wenn man bereit ist, kann man seine Änderungen mit anderen teilen, indem man sie in ein zentrales Repository hochlädt.

Quick Guide

Hier ist eine kurze Anleitung, um Git in einem neuen Projekt zu verwenden:

  1. Installation:

    2. Lade Git von git-scm.com herunter und installiere es. (Windows, macOS, Linux).

  2. Name und Email einrichten:

    3. git config --global user.name "Vor- und Nachname"
    git config --global user.email "beispiel@adresse.de"
  3. Repository erstellen:

    4. git init
  4. Status überprüfen:

    5. git status
  5. Datei hinzufügen:

    6. git add <Dateiname>
  6. Committen:

    7. git commit -m "Text für commit"
Begriffserklärung
  • Repository

    • Das Repository(Repo) ist ein Verzeichnis, in dem die Projekte und deren Versionshistorie gespeichert werden. Es handelt sich dabei um eine Art „Datenbank“, die alle Änderungen, die an einer Datei vorgenommen wurde, festhält.

  • Commits (Versionen)

    • Ein Commit ist ein Snapshot des aktuellen Zustands eines Projekts, der alle Änderungen speichert und es ermöglicht, die Historie der Entwicklung nachzuvollziehen und zu einem früheren Zustand zurückzukehren.

  • Merge

    • Ein Merge ist der Prozess, bei dem die Änderungen aus einem Branch in einen anderen Branch integriert werden, um die Entwicklung zusammenzuführen und die Historie der Änderungen zu vereinen.

  • Branches (Entwicklungszweige)

    • Beim Branch wird ermöglicht, verschiedene Entwicklungsstränge innerhalb eines Projekts zu erstellen, sodass Änderungen unabhängig und parallel getestet und entwickelt werden können, ohne die Hauptversion zu beeinträchtigen.

Szenarios
  • Ein Repository in der Befehlszeile erstellen, committen und evtl. auf GitHub stellen:

    • mkdir mein_projekt
    cd mein_projekt
    echo "# Text" >> README.md
    git init
    git add README.md
    git commit -m "Mein Erster commit"
    git push <URL z.b. von GitHub>
    git status

  • Bestehendes Repository klonen:

    • git clone <URL z.b. von GitHub>
  • Branch erstellen oder wechseln:

    • git branch feature-branch

    git checkout feature-branch
  • Zurück zum Haupt-Branch wechseln:

    • git checkout main
  • Branch mergen:

    • git checkout main
    git pull origin main
    git merge feature-branch
    git push origin main
Funktionen

Der Befehl git --help zeigt die Hilfe für Git an. Er öffnet die Dokumentation zu Git in einem Texteditor oder im Terminal. Du kannst auch spezifische Befehle nachschlagen, z.B. git commit --help, um Hilfe zu dem commit-Befehl zu erhalten.

git --force ist ein Befehl, der in Git verwendet wird, um eine Aktion gegen die normalen Sicherheitsvorkehrungen durchzusetzen. Der häufigste Anwendungsfall für --force ist beim Befehl git push, also beim Hochladen von Änderungen zu einem Remote-Repository.
ACHTUNG Die Verwendung von --force kann gefährlich sein, da sie dazu führen kann, dass die Arbeit anderer Teammitglieder verloren geht, wenn du ihre Commits überschreibst.
